# claude-token-roi

A single-page editorial guide and interactive calculator for estimating Claude Code token savings. Presents seven ranked optimization tips with impact / complexity / ROI scores and a live calculator that compounds the savings of adopted tips against the user's own usage profile.

**Live:** https://claude-token-roi-production.up.railway.app
**Repo:** https://github.com/ijayklein/claude-token-roi

## Stack

- **Frontend:** React 19 + Vite 7 + TypeScript, Tailwind v4, shadcn/ui (Radix), Recharts, Wouter (routing), Framer Motion
- **Backend:** Express 4 — production-only static server for the built SPA (no API routes)
- **Build:** `vite build` for the client + `esbuild` bundle of `server/index.ts` to `dist/index.js`
- **Package manager:** pnpm 10 (lockfile committed; `wouter@3.7.1` patched via `patches/`)
- **Hosting:** Railway (Nixpacks builder, config in `railway.toml`)

## How it works

`client/src/pages/Home.tsx` owns the content and the calculator. The `TIPS` array (7 entries) drives the tip cards, the summary table, and the bar chart.

**Scoring model (frequency-adjusted ROI).** Each tip carries:

- `savingsPerOcc` (1–10) — token savings each time the habit is applied
- `frequency` (1–10) — how often the situation arises in real usage
- `effectiveImpact = savingsPerOcc × frequency / 10`
- `complexity` (1–10) — adoption friction
- `roi = (savingsPerOcc × frequency) / complexity` — drives the table sort and bar chart
- `scope` — `"per-message"`, `"per-session"`, or `"per-project-stage"` (rendered as a colored badge via `SCOPE_COLORS`)

**Calculator.** Takes four inputs — messages/day, avg tokens/message, $/1M tokens, working days — and computes:

1. `baseMonthlyTokens = messagesPerDay × avgTokensPerMsg × workingDays`
2. For each adopted tip, applies a reduction factor (`TIP_REDUCTIONS = [0.35, 0.25, 0.15, 0.10, 0.08, 0.05, 0.03]`) compounded multiplicatively, not summed
3. Renders animated savings figures (tokens, $, %, optimized cost)

## Local development

```bash
pnpm install
pnpm dev      # vite --host on :3000
pnpm build    # vite build + esbuild server -> dist/
pnpm start    # NODE_ENV=production node dist/index.js, serves dist/public on $PORT (default 3000)
pnpm check    # tsc --noEmit
```

## Deployment

Railway runs `pnpm install --frozen-lockfile && pnpm run build`, then `pnpm run start`. Healthcheck is `GET /`. Pushes to `main` trigger redeploys; `/deploy-website update` from this directory commits, pushes, and runs `railway up --detach`.

## Environment variables

Optional. The site works fully without them.

- `VITE_ANALYTICS_ENDPOINT` — Umami host (e.g. `https://analytics.example.com`)
- `VITE_ANALYTICS_WEBSITE_ID` — Umami site id

If both are unset, `client/index.html` will still emit a `<script src="%VITE_ANALYTICS_ENDPOINT%/umami">` literal that 404s harmlessly. Remove the script tag in `client/index.html` if you don't plan to use analytics.
